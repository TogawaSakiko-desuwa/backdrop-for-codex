# 威胁模型

状态：基线更新（2026-07-23）

本文描述 Backdrop for Codex 的安全边界、主要威胁和剩余风险。它是工程决策基线，不是安全认证，也不能替代代码审计和运行时防护。

## 范围与安全目标

范围内组件包括 Windows 伴侣进程、托盘/设置、Store/MSIX Codex 发现与启动、回环 CDP 客户端、文件输入绑定与注入层、租约清理、过渡期回环 lease 服务、发布流水线和分发物。

安全目标：

- 只连接当前用户的官方 Store/MSIX Codex 主工作窗口；
- CDP 只经回环地址通信，过渡期 lease 服务也绝不形成远程入口；
- 只把用户明确选择、经路径规范化和文件头/容器签名校验并由只读 lease 锁定的单个媒体暴露给 Codex 渲染进程；
- 不修改、放宽或绕过 Codex CSP，媒体只使用页面 CSP 原生允许的 `blob:` URL；
- 不读取或持久化聊天，不上传本地媒体、设置或使用数据；
- 不修改 Codex 包，不提升权限，不在伴侣失联后遗留注入资源；
- 让用户能够验证发行物来源、散列和依赖清单。

非目标：抵御已控制当前用户、管理员或内核的攻击者；修复 Codex、Windows、浏览器媒体解码器或 .NET 自身漏洞；支持非 Store/MSIX Codex、非 x64 或非 Windows 11 环境。

## 资产

- Codex 会话的机密性和完整性，包括页面中可见但本项目不需要的聊天与账号状态；
- 当前 Windows 用户的文件、媒体路径、设置和进程信息；
- Codex 与 Backdrop for Codex 进程的可用性；
- 注入层和选择媒体的完整性；
- 源码、依赖、CI 凭据、发布标签、发布物、SBOM 和 attestations 的完整性；
- 用户对“无遥测、仅回环、不改包”的信任。

## 信任边界与数据流

1. **用户 → 设置/媒体选择**：用户选择本地文件，宿主在本地 JSON 中保存当前绝对路径及最多 8 条最近路径。
2. **伴侣 → Codex 发现**：伴侣识别受支持 MSIX 包、进程和主工作窗口。错误识别可能把控制发往非预期目标。
3. **伴侣 ↔ CDP 回环端点**：CDP WebSocket 可注入和清理表现层。它同时具备观察、执行和控制页面的更广泛能力。
4. **伴侣 → CDP 文件输入 → Codex 渲染器**：伴侣把已校验绝对路径经本机 CDP 绑定到本项目拥有的隐藏文件输入；页面只看到 `File` 内容和浏览器提供的文件名、大小、MIME type、修改时间，看不到完整绝对路径，并从该 `File` 创建 `blob:` URL。
5. **伴侣 ↔ 过渡期 lease 服务**：当前 `LoopbackMediaServer` 仍可能在 `127.0.0.1` 上生成随机 256 位令牌并持有只读 lease，但 endpoint/token 不注入 DOM，渲染器不通过它读取媒体；计划抽取为不监听端口的 lease-only 组件。
6. **伴侣 ↔ 注入层租约**：心跳/租约用于检测失联；清理移除媒体 `src`、撤销 `blob:` URL，并只删除本项目拥有的注入节点和样式。
7. **GitHub/NuGet → CI → 用户**：第三方源码、Actions 和依赖进入构建；发行版输出 ZIP、SHA-256、SPDX SBOM 与 GitHub attestations。

回环套接字不是 Windows 用户或进程隔离边界。同一用户下的任何进程通常都能尝试连接本地端口。CDP endpoint 一旦从命令行、日志、内存或进程检查接口泄露，就可能被使用直到 Codex 退出。过渡期媒体令牌不再进入 DOM，但同用户内存观察者仍可能取得它并在服务停止前重放。

## 攻击者假设

考虑以下攻击者：

- 同一 Windows 用户会话中的低权限恶意或被攻陷进程；
- 可诱导用户选择恶意/畸形媒体或设置文件的攻击者；
- 可提交恶意贡献、依赖更新、Action 更新或污染构建输入的供应链攻击者；
- 当端口被错误绑定、代理或转发时可访问主机的网络攻击者；
- 能读取公开日志、Issue、截图或误上传诊断包的机会型攻击者。

不假设能够抵御已经获得当前用户任意代码执行、管理员、调试权限、浏览器进程注入或内核控制的攻击者；这类攻击者本来就能访问大量相同资产。不过，本项目仍应避免降低其成本或扩大持久性。

## 主要威胁与控制

| 威胁 | 影响 | 主要控制 | 剩余风险 |
| --- | --- | --- | --- |
| 同一用户进程抢占/连接 CDP | 读取聊天、执行页面代码、操纵 Codex | 仅 `127.0.0.1`、验证端点/进程/包身份、日志不记录端点、向用户明确提示端口寿命 | **高**：回环无进程级身份验证；端口由 Codex 持有并持续到 Codex 完全退出 |
| CDP 或过渡期 lease 服务意外远程暴露 | 远程控制或读取当前媒体 | 禁止 `0.0.0.0`/局域网绑定；拒绝非回环 WebSocket URL；不提供端口转发；媒体 endpoint/token 不注入 DOM | **低至高**：取决于实现/部署是否保持不变量；移除过渡监听器后媒体侧风险将进一步缩小 |
| 连接错误进程或错误窗口 | 操纵非目标应用，泄露数据 | 校验 Store/MSIX 完整包名、激活 PID、进程启动时间、当前 Windows 会话、监听器所有权、目标元数据和主窗口选择；失败时关闭 | **中**：Codex 更新可能改变进程模型 |
| 过渡期媒体令牌被猜测或泄露 | 当前媒体被本机其他进程读取 | 每次启动 CSPRNG 256 位令牌、单段能力 URL、无目录列表、日志脱敏、不注入 DOM、退出失效 | **低**：同用户内存观察者仍可取得令牌；lease-only 重构完成前监听面仍存在 |
| 路径穿越、校验后替换或任意文件读取 | 读取用户未选择的文件、向解码器提供未校验内容 | 只接受用户明确选择的规范化绝对路径；扩展名与文件头/容器签名一致性校验；保持只读 lease；CDP 只绑定该文件；页面不接收完整路径 | **低/中**：已持有兼容共享句柄的同用户进程或复杂重解析点仍属剩余风险 |
| CSP 放宽或绕过 | 扩大任意页面资源加载/脚本执行面，破坏 Codex 安全策略 | 不修改 CSP，不使用或建议 CSP bypass；媒体经受审页面原生允许的 `blob:` URL 加载；加载失败即清理并报错 | **低/中**：Codex 更新可能改变 CSP；实现必须继续失败关闭而不能自动放宽策略 |
| HTML/JS/CSS 注入 | 在 Codex 页面执行攻击者内容 | 注入脚本固定化；数据经结构化序列化；不拼接路径/文件名为代码；只操作带 owner/generation 的 DOM；媒体文件通过 CDP 文件输入绑定 | **中**：CDP 本身允许脚本执行，审查缺陷仍可能扩大能力 |
| 畸形图片/视频利用解码器 | Codex/Chromium 崩溃或代码执行 | 限定格式、用户明确选择、依赖受支持系统/Chromium 解码器、及时更新 Codex/Windows | **中/高**：媒体解析器不在本项目控制内 |
| 设置文件篡改或资源耗尽 | 崩溃、错误路径、CPU/GPU/磁盘消耗 | schema、枚举和数值范围校验；最近路径数有界；原子写入；发现/清理超时；失败关闭 | **中**：同用户可修改本地状态；当前没有媒体文件大小或帧率上限，视频可能持续耗电 |
| 租约清理失效 | 背景残留、blob/文件句柄泄漏、意外持续控制 | 幂等清理、超时租约、导航/重连处理、移除 `src`、撤销 blob、只删除自有节点、退出清理、生命周期审查 | **中**：进程硬终止或 Codex 行为变化可能跳过部分主动清理；页面 lease 是最终恢复路径 |
| 权限提升或持久化误用 | 扩大系统影响 | 标准用户运行；不写 Codex 包/系统目录；不安装驱动/服务；自动启动需透明可撤销 | **低/中**：Windows 启动项和用户目录仍可被同用户修改 |
| 日志/Issue 泄露隐私 | 路径、令牌、聊天或账号信息公开 | 不记录聊天/令牌/媒体路径；媒体访问日志默认关闭；报告模板要求脱敏 | **中**：截图、转储和第三方异常可能绕过脱敏 |
| 依赖或 CI 供应链受损 | 恶意发行物获得 CDP 权限 | Actions 固定完整 SHA、Dependabot、CodeQL、最小权限、评审、SHA-256、SPDX SBOM、GitHub attestation | **中**：NuGet/维护者账号/构建平台仍是信任根；当前发布物可能无 Authenticode 签名 |
| 更新后 DOM/协议/CSP 漂移 | 错误注入、崩溃或安全检查失效 | 支持版本门控、目标验证、选择器与媒体加载失败关闭、兼容性审查、快速撤回 | **中**：Codex 是独立更新的外部产品 |
| 首次启动主页面尚未挂载 | 误报成功、遗留半注入资源 | 最长 10 秒有界轮询；只在验证页面完成媒体加载后报告成功；失败清理准备态节点 | **低/中**：极慢设备可能超时，需要重试而非放宽校验 |

## 必须保持的实现约束

- 所有监听器显式绑定 IPv4 回环 `127.0.0.1`；不要依赖“localhost”解析，也不要自动回退到任意地址。
- 对发现到的调试 URL 解析后验证地址类别；拒绝非回环、用户信息、异常 scheme 和重定向。
- 用户媒体必须来自明确文件选择，规范化为绝对路径，并同时校验扩展名、文件头/容器签名和长度；校验后由只读 lease 保持到停用或替换。
- 不得修改或绕过 Codex CSP。不得把过渡期媒体 endpoint/token 注入 DOM；页面媒体必须由 CDP 文件输入绑定并经 CSP 原生允许的 `blob:` URL 加载。
- 页面可见数据限于文件内容以及浏览器提供的文件名、大小、MIME type、修改时间；不得主动把完整绝对路径写入脚本、DOM、URL 或日志。
- 清理必须移除媒体 `src`、停止视频、撤销 `blob:` URL，并按 owner/generation 只移除本项目拥有的节点和样式。
- 过渡期媒体 URL 的令牌继续使用密码学安全随机源，每次进程启动轮换，不写磁盘、不写日志；服务只能绑定 `127.0.0.1` 并映射当前单文件。计划重构为不监听端口的 lease-only 组件，但在完成前不得误称监听器已经移除。
- 注入数据使用 JSON/DOM API 传递；不得把用户字符串插入可执行 JavaScript、HTML 或 CSS 源码。
- 设置输入按 schema、枚举和数值范围验证；写入使用同目录原子替换；损坏设置不会被执行，界面退回安全默认值。
- 不请求管理员权限，不修改/重签 Codex 包，不禁用 Windows 安全控制。
- CI 默认只读；发布 job 才获取 `contents: write`、`id-token: write` 和 `attestations: write`。

## 公开安全审查清单

安全相关变更的设计与代码评审至少应核对：非回环地址拒绝、错误包/进程拒绝、文件头/容器签名、长度与 lease 校验、CDP 文件输入只绑定明确选择的规范化路径、页面不暴露完整路径、`blob:` 加载成功判定、不使用 CSP bypass、媒体失败/导航/崩溃/硬退出后的 `src`/blob/自有节点清理、首次启动有界等待、日志脱敏、损坏设置和已实现的边界。过渡服务尚存时还要核对令牌熵与启动轮换、任意路径不可达、方法与 Range 边界。PR 必须记录审查结论、实际执行的构建/发布命令和未验证项；GitHub CI 执行 Release 构建、单文件发布形态检查与 CodeQL 分析。发布前还应验证 ZIP 的 SHA-256、SBOM 与实际输出一致，并使用 `gh attestation verify` 验证来源。

## 已知剩余风险与用户建议

最重要的剩余风险是 **CDP 对同一用户进程缺少强隔离**。即使本项目实现完全符合设计，其他本地软件也可能探测调试端口；来源不明的 Backdrop for Codex 构建则可能直接滥用 CDP。关闭壁纸或退出伴侣不能撤销 Codex 的启动参数。只在可信设备和普通用户会话中运行，保持 Windows 与 Codex 更新，不暴露/转发端口，验证发行物，使用完毕后完全退出 Codex。

如果使用环境要求对本地同用户恶意进程提供强防护，当前架构不能满足该要求，应避免使用。
