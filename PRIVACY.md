# 隐私说明

最后更新：2026-07-23

Backdrop for Codex 是在用户计算机上运行的开源桌面伴侣。本说明描述项目自身的设计；它不覆盖 OpenAI Codex、Microsoft Store、Windows、GitHub 或其他第三方服务的数据处理。

## 核心结论

- 不包含遥测、广告、用户画像或项目自有崩溃上报。
- 不运行项目自有云服务，不代理 Codex 与 OpenAI 的通信。
- 不以功能需要为由读取、保存或上传聊天内容。
- 媒体与设置在本机处理；设置包含绝对文件路径，因此可能泄露用户名、目录结构或文件命名信息。

## 本机处理的数据

### 设置

宿主把设置保存在 `%LOCALAPPDATA%\CodexWallpaper\settings.json`。其中 `CodexWallpaper` 是为避免升级后丢失设置而保留的内部兼容目录名，不是当前产品展示名称。当前设置模型会保存：

- 当前选中媒体的绝对路径；
- 最近使用的最多 8 条媒体绝对路径；
- 与背景显示和播放有关的本地偏好。

这些数据不会由本项目上传。能够以当前 Windows 用户身份读取该设置文件的其他进程，可能看到其中的路径。删除设置文件或使用应用提供的重置功能（若当前版本提供）可清除这些记录；卸载或删除程序目录不一定自动删除用户范围设置。

### 本地媒体

程序只在用户选择后访问 PNG、JPEG、WebP、MP4 或 WebM 文件。宿主规范化所选绝对路径，核对扩展名与对应的文件头/容器签名，并在使用期间保持只读 lease；因此普通编辑、替换或删除可能暂时被 Windows 拒绝，关闭或更换壁纸后句柄释放。

当前受审 Codex 页面 CSP 不允许从回环 HTTP 地址加载这些图片或视频。本项目不修改、不放宽也不绕过该 CSP。宿主经本机 CDP 把已校验、由 lease 锁定的文件绑定到页面内本项目拥有的隐藏文件输入，页面再生成 CSP 原生允许的 `blob:` URL。页面脚本在这段时间可以访问所选文件的内容，以及浏览器提供的文件名、大小、MIME type 和修改时间；它不能取得宿主传给 CDP 的完整绝对路径。宿主本身仍知道该路径，并按上一节所述保存设置。

当前代码仍保留 `LoopbackMediaServer` 作为兼容过渡层，以持有只读 lease；它可能短暂启动一个只绑定 `127.0.0.1`、使用随机 256 位令牌的单文件服务，但 endpoint 和令牌不再注入页面 DOM，Codex 渲染器也不再通过 HTTP 服务读取壁纸。后续计划把这部分收敛为 lease-only 组件。在完成前，令牌仍是本地能力凭据，不应进入日志或分享；回环限制也不能阻止同一用户会话中的恶意程序尝试访问本地端口。详见[威胁模型](THREAT_MODEL.md)。

### CDP 与 Codex 页面

项目通过回环 CDP 给已经验证的 Codex 主工作窗口添加表现层，并设置上述隐藏文件输入。预期功能不解析或持久化聊天内容，也不从页面读取媒体的完整本机路径。页面只保留当前 `blob:` URL 及其对应的媒体节点；关闭、更换、失联或 lease 到期时会移除 `src`、撤销 `blob:` URL，并删除本项目拥有的节点和样式。CDP 本身能够观察和控制页面，因而恶意或受损构建可能超出该用途。请验证发布来源，不要使用来源不明的二进制文件。

### 日志和诊断信息

程序可能生成本地运行状态、版本、连接阶段和错误信息，以便排错。项目自有的用户消息和日志设计上不记录聊天正文、过渡媒体服务令牌或媒体绝对路径；过渡媒体服务的默认 Kestrel 日志提供者被清空，以避免访问路径进入日志。底层系统异常的内部对象仍可能包含文件路径，因此不应公开上传异常转储。

错误、操作系统或第三方库仍可能产生未预见的信息。分享日志、截图、设置文件或转储前，请检查并删除用户名、绝对路径、聊天、令牌和其他敏感内容。内存转储可能包含完整进程内存，不应公开上传。

## 网络通信

项目自身正常运行所需的控制连接仅限本机回环 CDP。媒体内容通过页面内 `blob:` URL 读取，不经 HTTP 或外部网络。当前过渡实现还可能短暂启动只绑定 `127.0.0.1` 的单文件服务来持有 lease，但不会把其 endpoint/token 注入页面，也没有正常渲染请求需要访问它；它不应监听局域网/公网地址，也不应向项目维护者发送请求。退出伴侣会停止该过渡服务并释放 lease，但 Codex 自身的 CDP 端口会持续到 Codex 完全退出。

Codex 桌面应用会独立连接 OpenAI 服务，Windows、Microsoft Store、GitHub CLI 或操作系统组件也可能按各自设置联网。这些通信不是由本项目代理或控制，适用相应提供方的政策。

## 保留、删除和备份

本项目维护者不会收到上述本地数据，因此无法代用户访问、导出或删除。关闭壁纸或退出程序会移除媒体 `src`、撤销当次 `blob:` URL、删除自有页面节点，停止临时回环兼容服务并使当次令牌失效；设置会保留，直到用户删除上述本地设置文件。系统备份、企业漫游配置、取证工具或同步软件可能另行保留副本，需由用户或管理员管理。

首次成功增强启动后，程序会在当前用户桌面创建或更新 `Codex（动态背景）.lnk`，其参数为 `--launch`。删除该快捷方式可移除入口；取消设置窗口中的风险确认会阻止它下次自动增强启动。它不属于系统级启动项，也不会让程序随 Windows 自动启动。若伴侣已在托盘，第二实例只通过当前用户、当前 Windows 会话的本地命名管道转发显示或启动命令。

## 儿童、出售和共享

本项目不面向儿童收集数据，不出售个人信息，也不与维护者控制的广告商或数据经纪商共享数据，因为项目没有收集管道。

## 变更

如果未来功能需要遥测、远程服务或新增数据类别，必须先公开更新本说明与威胁模型，并提供明确、可撤回的选择；不得在普通维护更新中静默加入。版本历史可通过 Git 查看。

隐私或安全问题请按 [SECURITY.md](SECURITY.md) 私下报告。
