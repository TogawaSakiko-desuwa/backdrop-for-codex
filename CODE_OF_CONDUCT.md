# 贡献者行为准则

本行为准则改编自 Contributor Covenant 2.1。

本文件作为 Contributor Covenant 2.1 的改编文本，依照 [Creative Commons Attribution 4.0 International](https://creativecommons.org/licenses/by/4.0/) 使用和再分发；这项文档许可不改变项目代码的 Apache-2.0 许可。

## 我们的承诺

为了营造开放且友好的环境，我们承诺：无论年龄、体型、可见或不可见的残障、族裔、性别特征、性别认同与表达、经验水平、教育、社会经济地位、国籍、个人外貌、种族、宗教、性身份与性取向如何，参与本项目和社区对每个人而言都应是无骚扰的体验。

我们承诺以有助于形成开放、友善、多元、包容和健康社区的方式行事和互动。

## 行为标准

有助于营造积极环境的行为包括：

- 展现同理心和善意；
- 尊重不同意见、观点与经历；
- 得体地提供和接受建设性反馈；
- 承担责任，向受到影响的人道歉，并从经历中学习；
- 关注社区整体利益，而不只关注个人利益。

不可接受的行为包括：

- 使用色情化语言或图像，以及不受欢迎的性关注或挑逗；
- 挑衅、侮辱、贬损性评论，以及人身或政治攻击；
- 公开或私下骚扰；
- 未经明确许可发布他人的住址、邮箱等私人信息；
- 在专业环境中可合理视为不当的其他行为。

## 执行责任

项目维护者负责说明和执行可接受行为标准，并会对其认为不当、具有威胁、冒犯或有害的行为采取公平、适当的纠正措施。

维护者有权并有责任删除、编辑或拒绝不符合本准则的评论、提交、代码、Wiki 修改、Issue 和其他贡献，并在适当时说明处理理由。

## 适用范围

本准则适用于所有社区空间，也适用于个人在公共场合代表本项目或社区时。代表项目的情形包括使用官方账号、从官方邮箱发言、担任指定代表或参加线上/线下活动。

## 报告与执行

可通过仓库 **Security → Report a vulnerability** 私下报告辱骂、骚扰或其他不可接受行为；若该入口不可用，请使用维护者 GitHub 个人资料中列出的私密联系方式。不要在公开 Issue 中披露受影响者身份或敏感证据。

所有投诉都会被及时、公平地审查。负责处理者必须尊重报告者的隐私与安全；存在利益冲突时应回避。

## 执行指引

维护者将按行为的性质、影响和持续性决定后果：

1. **纠正**：对不专业或不受欢迎的语言/行为作出私下书面提醒，说明违反之处以及公开道歉是否适当。
2. **警告**：对单次或连续违规发出警告，并说明再次发生的后果；在指定时间内不得与相关人员互动，不得主动联系执行者。
3. **临时禁令**：在指定时间内禁止参与任何社区互动；违反禁令可能导致永久禁令。
4. **永久禁令**：对持续违规、骚扰个人或攻击某类人群者永久移除社区参与资格。
